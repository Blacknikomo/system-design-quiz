🥞 Stacks — LIFO in action

Last-in, first-out. Push, pop and peek are all O(1) at the top. Watch the valid-parentheses algorithm run on the stack, and see why recursion is a stack.
LIFO O(1) push / pop / peek top-only access call stack = a stack

A stack only lets you touch the top. push adds, pop removes, peek reads — all O(1) (it's just an array with a moving top index, no shifting). The discipline (last-in-first-out) is exactly what you need for nesting: matching brackets, undo/redo, expression evaluation, DFS, and backtracking.

The gotcha: recursion IS a stack — every call frame (locals + return address) is pushed onto the program's call stack, and deep recursion overflows it (StackOverflowError / segfault). The fix is to convert recursion to an explicit stack on the heap, which is far larger. Also know the monotonic-stack pattern: keep the stack sorted (e.g. decreasing) to solve next-greater-element / daily-temperatures in O(n) instead of O(n²).

What you're seeing

  1. Push adds a frame on top and bumps the top index — O(1).
  2. Pop removes the top frame; Peek reads it without removing — both O(1).
  3. Valid parens: scan left→right. An opening bracket is pushed.
  4. A closing bracket must match the top — if so, pop; if not (or stack empty), it's invalid.
  5. Valid ⇔ every close matched and the stack ends empty — O(n), each char touched once.
