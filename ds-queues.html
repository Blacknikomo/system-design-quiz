🎟️ Queues, Deque & Ring Buffer

FIFO in a fixed-size array. Watch a ring buffer enqueue at the tail and dequeue at the head, both indices marching forward with % capacity so the buffer wraps around — true O(1) at both ends.
FIFO ring / circular buffer O(1) enqueue & dequeue bounded capacity

A queue is first-in-first-out. A ring buffer implements it over a fixed array using two indices: head (next to read) and tail (next slot to write). Each op advances its index modulo capacity, so the logical sequence wraps past the end of the array back to slot 0. No element ever shifts — both ends are O(1). A deque generalizes this to push/pop at both ends.

The gotcha: using a plain JS array as a queue with arr.shift() is O(n) per dequeue — every remaining element gets re-indexed. That's a real, easy-to-ship performance bug; a ring buffer or linked list gives true O(1). Also: BFS uses a queue, DFS uses a stack, and bounded vs unbounded matters for backpressure — an unbounded queue under overload grows until it OOMs; a bounded one forces you to reject or block (apply backpressure).

What you're seeing

  1. Enqueue writes value at slot tail, then tail = (tail+1) % cap.
  2. Dequeue reads slot head, then head = (head+1) % cap.
  3. A count tracks size, distinguishing empty (count 0) from full (count = cap) when head==tail.
  4. On FULL: reject the new value, or overwrite the oldest (advancing head too).
  5. Indices wrap with % cap — the array end loops back to slot 0 (the "ring").
