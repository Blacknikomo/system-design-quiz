🔗 Union-Find — disjoint sets

Track which elements belong to the same group as a forest of trees. Watch union by rank keep trees flat, path compression flatten them further on every find, and cycle detection when two already-connected nodes are unioned.
amortized O(α(n)) union by rank path compression connectivity / cycle detection

A disjoint-set structure answers "are a and b in the same group?" in near-constant time. Each element points to a parent; the root of a tree is the group's representative. Union merges two groups by hanging the shorter tree's root under the taller's. Find walks parents to the root, then rewires every node it touched to point straight at the root.

The gotcha: you need both path compression and union by rank to reach near-constant O(α(n)); either alone is worse. It's add-and-merge only — there's no efficient delete or split. For undirected cycle detection it's the perfect tool, but directed cycles need DFS (back-edge / recursion-stack), not union-find.

Find(x)O(α(n)) amortized — α grows slower than log; effectively constant
Union(a,b)O(α(n)) amortized — two finds + one pointer flip
Connected(a,b)O(α(n)) — find(a) === find(b)
SpaceO(n) — parent[] + rank[]

What you're seeing

  1. Each element starts as its own root — 8 singleton trees.
  2. Union(a,b) finds both roots; if different, the lower-rank root hangs under the higher-rank root (tie → rank++).
  3. Find(x) walks parent pointers up to the root (highlighted), then compresses the path so every visited node points directly at the root.
  4. Group count drops by one on each real merge.
  5. If Union finds a and b already share a root, that's a cycle — the edge would close a loop.
